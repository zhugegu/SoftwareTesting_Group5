- 成员 D：递归/循环结构、异常与版本控制测试

- 成员 E：代码对象与跨版本/跨平台一致性测试

# 全局统一需求库（Master Requirements List）

|**主分类域 ID**|**子需求 ID**|**需求名称**|**详细描述**|**负责成员**|
|---|---|---|---|---|
|**R1：核心基准确定性**|**R1\.1**|核心等价确定性|在相同的 Python 版本和操作系统下，相同输入对象应产生完全一致的字节流哈希。|全体成员|
||**R1\.2**|跨操作系统一致性|相同版本的 Python 在 Windows、Linux、macOS 平台之间，常规有序类型序列化字节流 100% 相同。|成员 B / E|
|**R2：标量与整数边界**|**R2\.1**|跨平台字长无关性|大整数（超过 32 位/64 位字长）在不同位数的操作系统和架构下，序列化输出保持一致。|成员 A|
||**R2\.2**|变长编码边界稳定性|测试单字节、多字节存储临界点（如不同大小的整数在 marshal 中的变长编码边界）。|成员 A|
||**R2\.3**|内部优化隔离度|序列化输出不应受到 Python 解释器内部优化（如字符串/小整数 intern 机制）的影响。|成员 A|
|**R3：浮点与复数特殊值**|**R3\.1**|浮点特殊值常规确定性|浮点数中的 NaN, Inf, \-0\.0 在相同环境下其序列化输出具备单进程确定性。|成员 B|
||**R3\.2**|NaN 载荷跨平台变异性|跨操作系统与架构时，由于底层的硬件和 C 库差异，NaN 载荷位模式在序列化时的表现。|成员 B|
|**R4：复合容器与乱序集合**|**R4\.1**|有序容器确定性行为|在默认环境下，标准有序容器（list, tuple, 以及 Python3\.6\+ 的 dict）的序列化输出必须完全确定。|成员 C|
||**R4\.2**|集合类型随机非确定性|在默认随机种子（PYTHONHASHSEED=random）环境下，set 和 frozenset 的序列化表现出非确定性行为。|成员 C|
||**R4\.3**|确定性环境可控恢复性|当显式固定环境变量 PYTHONHASHSEED 为特定整数时，集合类型的序列化输出必须完全恢复确定性。|成员 C|
||**R4\.4**|非确定性传播污染效应|当有序容器内部嵌套了乱序集合时，集合的非确定性会向外层容器传播，导致整个容器序列化哈希随机化。|成员 C|
||**R4\.5**|极大容器内存边界稳定性|当容器（list, dict）规模达到百万级（10\*\*6）边界时，模块不崩溃且保持输出稳定一致。|成员 C|
|**R5：复杂结构与异常流**|**R5\.1**|循环引用安全检测能力|当输入包含循环引用的超深递归结构时，模块能稳定检测并抛出 ValueError，不引发 C 栈溢出。|成员 D|
||**R5\.2**|嵌套极限边界稳定性|在系统最大允许的嵌套深度临界点，合法的深层嵌套对象能够成功且稳定地序列化。|成员 D|
||**R5\.3**|不支持类型异常一致性|当传入非支持类型（如 generator, class instance）时，模块抛出异常的行为在多次运行中保持一致。|成员 D|
||**R5\.4**|共享引用输出确定性|针对同一个对象在复杂结构中被多次引用的场景（Shared References），序列化输出具备确定性。|成员 D|
|**R6：运行时环境与字节码**|**R6\.1**|编译优化级别隔离度|Python 不同的编译优化级别（如 \-O, \-OO）对生成的代码对象序列化结果产生的确定性影响。|成员 E|
||**R6\.2**|代码对象字段透明性|代码对象的序列化输出仅由其解构后的字段内容决定，不应受到内存分配地址或编译次序的影响。|成员 E|
||**R6\.3**|版本间格式变化可记录性|准确发现并文档化 marshal 格式在不同 Python 主版本（如 3\.10, 3\.11）之间的不兼容变化。|成员 E|

- `R2.1`：跨平台字长无关性 

- `R2.2`：变长编码边界稳定性（原 A 关注的 R3 转换为 R2\.2，防止与 C 冲突）

- `R2.3`：内部优化隔离度
