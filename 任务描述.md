# 任务描述

# 背景

`marshal` 模块实现了 Python 内部对象类型的序列化与反序列化。它主要用于读写 Python 模块的“伪编译”字节码（\.pyc 文件）。“封送”（Marshalling）将受支持的 Python 对象转换为二进制字节流，而“解封送”（Unmarshalling）则执行相反的操作。

# 任务描述

`marshal` 格式被设计为与体系结构无关（对于相同的 Python 版本，不同的操作系统应产生相同的数据），但它刻意不在不同 Python 版本之间保持稳定——格式可能会（也确实会）在主版本发布之间发生变化。即使在同一个版本内，浮点数处理、递归数据结构以及内部实现细节上的细微差异也可能导致非确定性的输出。

我想了解 `marshal` 模块的稳定性和正确性：相同的输入是否总是产生相同的（序列化）输出？

我们将“相同的输入和输出”定义为哈希相同（逻辑等价是不够的）。这意味着，在任何情况下，一个输入必须产生相同的 marshal 字节流。

需要研究的可能选项包括：

- 不同的操作系统（例如 Windows、Linux、macOS）

- 不同的 Python 版本（marshal 格式可能会变化）

- 浮点数的精度和特殊值（NaN、Inf）

- 递归和循环数据结构（例如包含自身的列表）

- 空集合和极大集合

请保持对其他选项的开放性。

为 `marshal` 模块的稳定性和正确性开发一个测试套件。

你应该使用课堂上学习到的黑盒测试技术（等价类划分、边界值分析、模糊测试等）。

由于 `marshal` 的源代码是可获取的（`Python/marshal.c` 和 `Lib/marshal.py`），你也可以应用课堂中讨论到的白盒测试方法（例如所有定义\-所有使用覆盖）。
